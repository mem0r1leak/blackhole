# 🌌 blackhole

WIP (work in progress)
A high-density data singularity engine. Sucking bytes into the event horizon and crushing them into a lighter reality. 🗜️💥

## 🛰️ Про проєкт

blackhole — це експериментальний двигун для стиснення даних, натхненний ідеями ентропійного кодування та низькорівневої магії С++.

## 🏗️ Як це працює (приблизно)

За допомогою алгоритмів на основі дерева Хаффмана довгі послідовності бітів кодуються в коротші. Процес виглядає десь так:

1. Аналіз маси: Вивчаємо, які байти зустрічаються найчастіше. 📊
2. Побудова сингулярності: Створюємо структуру, де найпопулярніші символи стають найлегшими. 🌲
3. Гравітаційний колапс: Перетворюємо вхідний потік у щільну бітову послідовність. ⚡


## ✨Features✨

1. **DANGER!!!** Проект використовує STL (vector, priority_queue, span, etc.)
2. Всі тести написані Gemini 🤓
3. Linux only


### How to build?

```bash
cmake -B build && cmake --build build
```


### 🎓 Освітня місія (The Learning Loop)

Цей проект — не спроба вбити WinRAR чи Zstd. Це пісочниця, створена виключно для того, щоб автор міг:
  1. Зрозуміти, як працюють бітові потоки, не зламавши собі мозок (спойлер: мозок постраждав).
  2. Розібратись як працює кодування Хаффмана.
  3. Попрактикувати C++23, поки він ще вважається "модним та молодіжним".

⚠️ Disclaimer: Автор поняття не має, що він робить. Усі збіги з реальними алгоритмами стиснення — чиста випадковість. Використовуйте на свій страх, ризик та ентропію.
Алгоритми можуть бути неправильно реалізовані чи не за теорією. Проект створений для навчання та дослідження роботи архіваторів.
Гуманізм: Ми запевняємо, що під час тестів жоден байт не був видалений несправедливо. Ми поважаємо права кожного біта на існування.
Відповідальність: Використовуючи blackhole, ви погоджуєтесь, що ваші дані можуть бути стиснуті до стану сингулярності, звідки немає вороття. Автор не несе відповідальності за втрату файлів, викривлення простору-часу або раптовий kernel panic.
